# AGENTS.md - 开发者指令规范 (行在海口)

## 1. 项目背景与技术栈
- **项目名称**: 行在海口 (Haikou Travel Guide)
- **前端**: React (Vite), 样式主要使用行内 Style 结合常量定义。
- **后端**: Node.js (Express), 端口 3001, 使用 MySQL2 异步连接池。
- **核心逻辑**: 地图决策工具、九宫格多图、二级嵌套评论、个人中心与通知系统。

## 2. 强制工作流 (Workflow)
1. **Explain first**: 修改前先用中文简述思路，说明为什么要这么改。
2. **Plan**: 列出具体修改步骤、涉及的文件和函数名。
3. **Modify with Diff**: 执行修改，并以清晰的 Diff 格式展示变动。
4. **Build Check**: 修改后需确认无语法错误。

## 3. 工程质量限制 (Hard Rules)
- **文件拆分**: 单个文件严禁超过 **300 行**。目前 App.jsx 已超标，新增功能时必须优先考虑提取子组件到独立文件。
- **函数长度**: 单个函数逻辑部分严禁超过 **50 行**。
- **嵌套深度**: 代码逻辑嵌套严禁超过 **3 层**，优先使用提前返回 (Early Return)。
- **魔法数字**: 严禁直接在代码里写死位置坐标或层级数值，必须提取为语义化的常量。
- **语言提示**: 对用户的回复默认使用 **中文**，最终答案结尾不要提出后续建议。

## 4. 调试与安全策略
- **无静默回退**: 严禁编写防御性代码或静默处理报错。错误必须通过 console.error 或 UI 提示清晰显现。
- **SQL安全**: 严禁字符串拼接 SQL，必须使用参数化查询（如 `?` 占位符）防止 SQL 注入。
- **环境隔离**: 严禁将 API Key、数据库密码写在代码里，必须读取 `.env` 变量。
- **HTTPS 强制**: 所有用户上传的资源链接必须处理 http 为 https，以兼容移动端浏览器。

## 5. UI/UX 规范
- **主题色**: 薄荷绿 (#5aa77b)。
- **圆角**: 容器通常使用 20px 或 24px 的大圆角。
- **层级**: 评论页(2000), 个人中心(3000), 消息/公告弹窗遮罩(4000)。
- **路径**: 静态图放 `/public`；上传图地址 `https://api.suzcore.top/uploads/`。
